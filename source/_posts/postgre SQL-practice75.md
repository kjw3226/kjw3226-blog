```
추천 시스템을 구축할 때는 어떤 지표를 확인하면 좋을까요? 웹사이트의 CTR, CVR, 추천을

통한 매출 등을 집계하는 것도 하나의 방법입니다.

하지만 이 방법들은 로직과 관련한 직접적인 지표가 아니므로, 추천 시스템에서 어떤 부분을 

개선해야 좋을지 명확한 판단을 내리기 어렵습니다.


Microsoft Research 기술 리포트 중 샤니 가이(Shani Guy)가 작성한

'Evaluating Recommender(2009)'을 보면 추천 시스템을 구축할 때의 평가 방법, 평가 지표등이 

자세하게 나와 있습니다.  무료 라이센스이므로 관심이 있다면 참고하기 바랍니다.

리포트에서 설명하는 지표를 간단하게 정리해보면 다음과 같습니다.


<추천 시스템의 대표적인 평가 지표 목록>

1. Converage : 전체 사용자와 아이템 중에서 추천이 제공되는 사용장와 아이템의 비율을 나타내는 값

2. Confidence : (시스템적인)추천 아이템의 신뢰도를 나타내는 값

               어떤 아이템을 추천하기 위해 사용한 데이터양이 많으면 Confidence도 높음.

3. Trust : (사용자적인)추천 아이템의 신뢰도를 나타내는 값

          사용자에게 추천이 제대로 되는지 물어보는 방법으로 얻어냄.

4. Novelty : 추천된 아이템의 신규성을 나타내는 값

5. Serendipity : 뜻밖의 아이템을 추천하는지를 나타내는 값

6. Diversity : 추천된 아이템의 다양성을 나타내는 값

7. Utilty : 서비스 추천의 유익성을 나타내는 값

8. Risk : 리스크를 포함한 아이템(주식 등)을 추천할 때 고려해야 함.

9. Robustness : DOS 공격 등이 있을 때 추천 내용이 왜곡되지는 않는지 나타내는 값

10. Privacy : 추천 내용을 통해 개인 정보를 추측할 수 있는지를 나타내는 값

11. Adaptivity : 아이템이 업데이트와 유행 등의 변화에 잘 대응할 수 있는지 나타내는 값

12. Scalability : 데이터의 양이 늘어났을 때 대응할 수 있는지 나타내는 값
```

시스템적인 관심에서도 추천 시스템을 고려해야 하므로 여러 관점에서 살펴보기 바랍니다.
